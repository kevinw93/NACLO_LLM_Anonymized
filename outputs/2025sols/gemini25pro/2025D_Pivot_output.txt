An analysis of the problem reveals the core mechanic of a "pivot language." To translate a word `w_A` from language A to language C via pivot language B, the following process occurs:
1.  Identify the set of objects `O_A` that `w_A` can describe.
2.  For each object in `O_A`, find the corresponding word in the pivot language B. This gives a set of pivot words `W_B`.
3.  For each word in `W_B`, find all the objects it can describe in language B. The union of these object sets forms a new, potentially expanded, set of objects `O_B`.
4.  For each object in `O_B`, find its corresponding word in language C. This final set of words `W_C` represents all possible translations.

A "pivot translation" from word `w_A` to `w_C` is possible if `w_C` is in the set of potential outcomes `W_C`. This can be simplified: a path exists if there is an input object `o_in` (described by `w_A`) and an output object `o_out` (described by `w_C`) for which the pivot language uses the same word.
